    public withNode(string[] $nodes, string $uri, string $secret): self
    Setup Yate API node records for one host/server

    Setup Yate API node records for one host/server

    Usual, multiple yate 'nodes' may reside on one host and share one API entry point.

    Then, we need each api entry point to be registered in the config with a lost of 'nodes' and it's access secret.

    Mind that each 'node' may only be registered once, mean if you include a 'node', say, 'hss' in two cosecutive calls of the methid, only last record will survive.

    Parameters

    $nodes  

    Node's list of component installed, like ['hss','smsc','ucn']
    $uri  

    Node's api entry point, full URI like 'http://10.3.2.5/api.php'
    $secret  

    Node's secret to authorise access
